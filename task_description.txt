ORM Doctrine. Миграции. Связи. Внешние ключи. MyIsam vs InnoDB
Страницы:
- Список всех постов (главная страница): /posts
- Детальная страница поста /post/{id} (например /post/4 для отображения 4-го поста)
- Список постов пользователя: /user/{id}/posts
- Информация о пользователе: /user/{id}

Используем все что знаем об MVC:
- роутер для маршрутизации страниц (разрешается и приветствуется использование сторонней библиотеки, например https://github.com/auraphp/Aura.Router);
- twig для шаблонов страниц;
- doctrine для моделей, связей и миграций;
- Для оформления - bootstrap.

Шаги:
1. Создать сущность User и Post (поля определяем сами и связи тоже)
2. Создать миграции для создания таблиц и тестовых данных (несколько пользователей, и у каждого по несколько постов)
3. Создать страницу списка постов (только названия со ссылкой на продолжение поста)
4. На странице конкретного поста отображаем ссылку на автора статьи (User).
5. На странице автора статьи нужно отобразить все посты этого автора (пользователя).

Задания со звёздочкой (для самых отчаянных):
- Создать форму создания поста.
- Создать возможность удаления поста в списке.
